Trade & Investment Management Portal (TIMP) – Demo Walkthrough

Narrative-driven Demo-only Use as your speaking guide

1) Opening (30–45s)

“Today I’ll show a demo of TIMP — a single platform that turns roadshows/forums into an outcome-driven engine: registration → content access → matchmaking → deal tracking → reporting.”

Optional: start with Region = Non-Regional for global view. Then switch to a region to show segregation.

2) Public Portal (3–4 min)

  1. Go to Events → show annual calendar + event profiles.
  2. Open an event → click Register → show confirmation toast.
  3. Show “published materials” + “recordings” sections (post-event continuity).
  4. Go to Deal Room → show recommended matches + meeting requests.

3) Deal Facilitation (2–3 min)

  1. In Deal Room, open a deal → show recommended investors (sector + ticket size).
  2. Click Schedule 1:1 (demo) → meeting appears in list.
  3. Open Pitching → quickfire 60s timer + readiness checklist + “Request Prospectus”.

4) Admin Console (3–4 min)

  1. Admin Events Manager → create/publish an event (demo).
  2. Registrations → approve/reject.
  3. Deal Book → generate by country/sector (demo).
  4. Analytics → show engagement + downloads metrics.
  5. Archive → search institutional memory.

5) Close (30–45s)

“This prototype is the front-end experience. The real build will add SSO/RBAC, database, email automation, integration with CRM and Africa Trade Gateway, and audit-grade reporting.”

What to hand to developers

Use this UI as a reference. Then provide: feature backlog, user roles, data model (events, users, deals, investors, meetings), API list, and roadmap milestones.
